# 依赖管理目录

此目录包含所有依赖管理的配置文件。

## 配置文件说明

- `base.yaml`: 基础依赖配置，包含所有环境共用的依赖
- `development.yaml`: 开发环境依赖配置，包含开发工具和调试依赖
- `production.yaml`: 生产环境依赖配置，只包含运行时必需的依赖

## 依赖管理原则

1. 明确声明所有依赖版本
2. 定期更新依赖安全检查
3. 禁止使用已弃用的依赖版本
4. 使用虚拟环境隔离依赖

## 依赖更新流程

1. 检查依赖更新和安全公告
2. 在开发环境测试新版本
3. 更新依赖配置文件
4. 执行完整测试验证
5. 部署到生产环境
